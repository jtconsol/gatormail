GatorMail preferences. Preferences in GatorMail are implemented as {@link java.util.Properties} backed by whatever you want.

Below is a list of properties and their meaning. Unless otherwise stated it is assumed that each entry is a free form {@link java.lang.String}.

compose.replyTo
The address the user wants messages to be replied to. This must be in a form acceptable to {@link javax.mail.internet.InternetAddress#InternetAddress(String)}.
compose.signature
The user's signature.
compose.X-Image-Url
A URL to an image that represents the sender. The linked image should be about the size of a "buddy icon" though there is no known standard defining the size.
folder.list.threading
True if message threading should be turned on, else false. This value must be parseable by {@link java.lang.Boolean#valueOf(String)}.
folder.list.count
How many messages to list at once in the folder view. This must be parseable by {@link java.lang.Integer#parseInt(String)}.
folder.list.sort
The default message sort for the folder view.
message.junk.threashold
Any messages with a junk score greater or equal to this threashold will be flagged at junk. A value of zero means the junk threashold is disabled. Default is 8. This must be parseable by {@link java.lang.Integer#parseInt(String)}.
session.maxInactive.multiplier
This is used to adjust the user's inactive session experiation time by multiplying the container's default value by this number at login time. Default is 1.0. This must be parseable by {@link java.lang.Float#parseFloat(String)}.
user.login.last
The time the user last logged in.
user.name
The name of the user as she would like it displayed. This should not contain any new line characters.
view.header.hide
True if the page header's should be hidden. This value must be parseable by {@link java.lang.Boolean#valueOf(String)}.
